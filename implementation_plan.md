# HackIt Room — MVP 實作計畫

## 概述

在 3–7 天內交付一個可運行的 HackIt Room MVP：
一個帶有神秘氛圍的互動式網頁空間，使用者能探索物件、閱讀隨機敘事片段、留下想法，並感受到「世界正在回應」的非線性回饋。

---

## User Review Required

> [!IMPORTANT]
> **技術選型決策：Godot vs 純 Web**
>
> 計畫書提到使用 Godot Engine（HTML5 匯出），但 MVP 階段有一個替代方案值得評估：
>
> | 方案 | 優勢 | 劣勢 |
> |------|------|------|
> | **Godot Engine** | 像素風格更精準、動畫控制強、未來擴展性高 | 需要安裝 Godot、匯出設定複雜、開發週期較長 |
> | **純 HTML/CSS/JS（Canvas/SVG）** | 立即可開發、部署簡單、無需額外工具 | 2D 場景效果需手動實作 |
>
> **建議**：MVP 使用純 Web 技術（HTML Canvas + CSS 動畫），先驗證體驗核心，之後再遷移至 Godot。
> 請確認你偏好哪個方向。

> [!NOTE]
> **後端部署位置**
> MVP 的後端 Node.js API 將在本機運行。如需雲端部署（Vercel、Railway 等），請告知。

## Open Questions

1. **視覺風格**：像素風（8-bit）？或是現代暗黑風（glassmorphism + 粒子效果）？
2. **互動物件內容**：5 個物件分別是什麼？（例：攤位、收音機、紙張、門、...？）
3. **語言**：UI 與敘事內容以中文為主？還是中英混合？
4. **「異常感」的定義**：你希望使用者進入後 30 秒內感受到什麼？環境音？畫面扭曲？文字行為異常？

## Proposed Changes

### Frontend — 房間場景

#### [NEW] index.html
進入頁面：全黑畫面 → 緩慢淡入房間，無任何說明文字，只有一句模糊的引導語（或完全沒有）

#### [NEW] room.js — Canvas 2D 場景
- 房間底圖（SVG 或 Canvas 繪製）
- 5 個互動物件，各自有 hover 光暈效果
- 點擊後觸發：文字浮現動畫 + 隨機敘事內容
- 環境粒子（灰塵浮動 / 燈光閃爍）

#### [NEW] style.css — 暗黑氛圍設計系統
- 主色：深墨色背景（`#0a0a0f`）
- 強調色：霧藍 / 琥珀黃（模糊燈光感）
- 字體：等寬字體（神秘終端機感）

---

### Frontend — 互動物件（5 個）

| 物件 | 初始狀態 | 點擊後 |
|------|----------|--------|
| 📻 收音機 | 微微閃爍 | 播放雜訊 + 顯示「截取到的訊號」 |
| 📄 紙張（散落） | 隨機旋轉 | 顯示其他人留下的想法片段 |
| 🚪 門 | 輕微震動 | 顯示「尚未開放」或神秘倒數 |
| 💡 燈泡 | 忽明忽暗 | 觸發房間光線變化 + 顯示一句話 |
| 🗃️ 抽屜 / 箱子 | 靜止 | 打開動畫 + 顯示輸入框（留下想法） |

---

### Frontend — 輸入系統

- 只在「抽屜」互動後出現
- 極簡輸入框：無標題、無提示、無送出按鈕文字（只有一個 `→`）
- 送出後：文字緩慢消失 → 「已記下」的感覺（不確認、不評分）

---

### Backend — Node.js API

#### [NEW] server.js
```javascript
// GET  /messages  → 隨機返回 3–5 條儲存的訊息
// POST /messages  → 儲存新訊息（附上時間戳）
```

- 初期使用 JSON 檔案儲存（`data/messages.json`）
- 內建種子資料（10–20 條預設敘事片段）

## 核心體驗設計（30 秒異常感）

進入後的時間軸：

```
00:00  全黑畫面，無任何 UI
00:02  音效：遠處隱約的環境音（低頻嗡嗡聲）
00:04  房間輪廓緩慢淡入
00:08  物件一個一個「醒來」（燈光閃爍先出現）
00:15  收音機開始微微閃爍
00:20  紙張隨機飄動（CSS animation）
00:25  畫面邊緣出現極淡的掃描線效果
00:30  ← 使用者此時應感到「這個空間有點奇怪」
```

## Verification Plan

### 自動測試
- `npm test` 驗證 API routes（GET/POST /messages）
- 前端靜態服務：`npx serve frontend/`

### 手動驗證
- 在瀏覽器中開啟並計時：30 秒內是否感受到「異常感」？
- 測試所有 5 個互動物件的點擊行為
- 測試輸入 → 儲存 → 下次讀取能看到內容
